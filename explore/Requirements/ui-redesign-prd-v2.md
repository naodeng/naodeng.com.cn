# UI Redesign PRD v2

## 背景与参考
本轮 UI Redesign 以以下站点为视觉与信息架构参考：
- `https://astro.build/`
- `https://docs.astro.build/en/getting-started/`
- `https://astro.new/latest/`

抽取的可复用模式：
- 顶部信息密度高但结构清晰（一级导航 + 资源入口 + 明确 CTA）。
- Hero 区块突出“核心价值 + 立即行动”。
- 文档阅读页强调“左侧导航 + 右侧目录 + 中间正文”的稳定阅读流。
- 资源卡片统一视觉语言（标题、说明、操作按钮、状态/标签）。
- 明确的“下一步动作”链路（Get Started / Docs / Templates / Community）。

## 目标
- 在不推翻当前 IA 的前提下，建立统一视觉系统，提升站点专业感与辨识度。
- 强化“内容发现 -> 深度阅读 -> 继续探索”的路径效率。
- 保持 `en` / `zh-cn` 双语体验一致，避免仅中文或仅英文页面视觉分裂。
- 保障性能和可访问性不退化。

## 已确认决策
- 视觉风格强度：`偏文档克制`（信息优先、装饰弱化）。
- 首页首屏文案：`2-3 行摘要 + 主/次 CTA`，同时在首屏下方增加“信息补充模块”提升内容丰富度。
- 卡片封面策略：`弱化封面（信息优先）`，封面只承担辅助识别，不主导信息层级。
- 发布策略：`开发分阶段 + 一次性发布`（全量页面同版本切换）。
- 广告密度规则：
  - 每个页面至少 1 个广告位；
  - 列表页每屏最多 1 个广告；
  - 详情页正文中部最多 1 个广告。
  - 广告不可插入在标题与首段正文之间；
  - 广告容器需要预留固定占位，避免内容跳动；
  - 广告视觉层级低于正文与导航（弱边框、弱背景、不抢主色）。
  - 例外页可豁免“每页至少 1 个广告位”：`404`、`隐私/条款`、正文高度不足 800px 的短内容页。

## 范围
- 页面范围：
  - 首页（`/[lang]/`）
  - 博客列表与详情（`/[lang]/blog/*`）
  - Wiki 首页与详情（`/[lang]/wiki/*`）
  - AI Wiki 首页与详情（`/[lang]/AIWiki/*`）
  - Guild 首页与详情（`/[lang]/guild/*`）
- 系统范围：
  - Design Tokens
  - 排版系统
  - 组件层（卡片、目录、meta、广告槽、相关词条、代码块工具条）
  - 交互层（筛选、搜索、目录高亮、滚动反馈）

## 非范围
- 重写内容正文。
- 引入新前端框架（保持 Astro + 现有技术栈）。
- 重做 Header/Footer 信息架构（仅视觉优化和可用性微调）。

## 用户与场景
- 新访客：快速理解站点价值，快速进入“博客/Wiki/AIWiki/Guild”。
- 深度阅读用户：长文阅读、目录导航、相关词条跳转、代码片段复制。
- 搜索导流用户：直接落地详情页后可继续探索同主题内容。

## Research & Validation Plan
- 研究目标：
  - 验证首页首屏信息密度是否更易理解；
  - 验证详情页 TOC 与相关词条是否提升连续阅读；
  - 验证广告低干扰策略是否成立。
- 研究方法：
  - 可用性测试（任务制）：每语言 5-8 人；
  - 行为分析：滚动深度、二跳率、关键 CTA 点击；
  - 定性访谈：上线后 2 周回访核心用户。
- 任务脚本（示例）：
  - 30 秒内找到 AIWiki 入口并进入词条页；
  - 在详情页通过 TOC 跳转到指定章节；
  - 从相关词条继续阅读第二篇内容。
- 输出物：
  - 可用性问题清单（按 P0/P1/P2）；
  - 指标对比报告（上线前后 28 天）；
  - 迭代建议（下一版本 backlog）。

## 功能与体验需求
### 1. 全局视觉系统
- 统一色板、间距、圆角、边框、阴影、动效节奏。
- 统一标题层级（H1-H4）与正文节奏（段间距、列表、引用、代码）。
- 统一按钮语义（主按钮/次按钮/幽灵按钮）与可点击反馈。
- 控制视觉强度：弱阴影、弱渐变、弱动效，避免“营销页”式强装饰。

### 2. 首页改造
- Hero 区块：采用 2-3 行摘要，保留品牌定位并强化第一屏主/次 CTA。
- 在 Hero 下方新增“信息补充模块”（例如：能力点、专题入口、最新动态、关键数据）以补足内容丰富度。
- 新增“快速入口区”：Blog / Wiki / AI Wiki / Guild 四象限。
- 博文与词条卡片：统一视觉规范、降低封面干扰、提升信息可扫读性。
  - 封面高度降低，遮罩与视觉对比减弱，优先保障标题与摘要可读性。
- 区块间加入“继续探索”引导（类似 Astro 的资源链路）。

### 3. 详情页阅读体验
- 右侧目录：可见范围限定在正文高度内，避免与页脚/评论区冲突。
- TOC 高亮：滚动时稳定命中当前章节，支持 h2/h3。
- 作者卡与 meta 区：高度更紧凑，保留原信息。
- 相关词条：博客/Wiki/AIWiki/Guild 详情页统一模块样式和交互。

### 4. 文档化与资源化体验
- Wiki / AIWiki 列表：参考 docs/template 风格，强调“可浏览 + 可筛选 + 可跳转”。
- 卡片操作语义统一：标题、摘要、标签、操作入口布局一致。
- 目录与侧栏在移动端可折叠，桌面端固定阅读流。

### 5. 可访问性
- 焦点可见（focus-visible）。
- 键盘可达（目录、筛选、搜索、分享按钮）。
- `prefers-reduced-motion` 兼容。
- 语义结构（landmark、heading 层级、aria-current）。

### 6. 性能
- 首屏不新增重型依赖。
- 图片与封面继续优化（避免 CLS）。
- 控制脚本注入策略（生产域名白名单、非必要不加载）。
- 卡片封面采用轻量化策略（优先渐变/低成本占位，避免重图导致首屏抖动）。
- 广告加载与布局需满足密度规则，避免影响主阅读流与滚动稳定性。

## 指标
- 首页核心入口 CTR：+15%（对比上线前 28 天基线）。
- 详情页滚动深度：+20%（对比上线前 28 天基线）。
- Wiki/AIWiki 二跳率：+10%（对比上线前 28 天基线）。
- Lighthouse（移动端）：
  - Performance >= 90
  - Accessibility >= 95
  - Best Practices >= 95
  - SEO >= 95
- 数据口径：
  - 基线周期：上线前连续 28 天；
  - 评估周期：上线后连续 28 天；
  - 数据来源：Counterscale + E2E/Lighthouse 报告。

## Measurement Spec
- 首页核心入口 CTR：
  - 定义：首页首屏核心入口区点击 / 首页 UV；
  - 来源：页面事件埋点或点击日志聚合。
- 详情页滚动深度：
  - 定义：达到 25/50/75/100% 的会话占比；
  - 来源：滚动事件埋点或页面分析工具。
- Wiki/AIWiki 二跳率：
  - 定义：进入词条页后继续访问同频道第二页的会话占比；
  - 来源：站内路径分析与 referrer 链路。
- 性能与可访问性：
  - 定义：Lighthouse 移动端四项得分 + Web Vitals；
  - 来源：CI 报告与抽样实测。

## 验收标准
- 视觉验收：5 类核心页面均符合新 token 与排版规范。
- 交互验收：TOC、筛选、复制、相关词条、广告槽位均正常。
- 回归验收：E2E 关键路径通过；无 P1/P2 视觉或功能回归。
- 广告验收：
  - 正文阅读路径不被切断（首屏与段落主链路连续）；
  - 广告不引发可感知 CLS 波动（页面 CLS <= 0.1）；
  - 广告区点击误触率无明显上升。
- 可访问性验收：
  - 文本对比度 >= 4.5:1；
  - 触控目标 >= 44x44px；
  - 移动端正文最小字号 >= 16px。

## 风险与缓解
- 风险：样式改动面大，局部页面易出现回归。
  - 缓解：分阶段开发（先组件后页面）；发布日一次性切换；附带回归截图与 E2E。
- 风险：移动端折叠交互复杂导致可用性下降。
  - 缓解：移动端优先做“可用优先”，再做动效增强。

## Exception Policy
- 广告规则例外判定：
  - `404`、`隐私/条款`：允许无广告；
  - 短内容页（正文高度 < 800px）：允许无正文中部广告；
  - 当广告影响可访问性或 CLS 超阈值时，允许临时降级广告位。
