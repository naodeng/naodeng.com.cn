系列建议写法：

- 先写没有该工具或方案之前，这项测试工作通常是怎么做的。
- 再写引入新工具或方案之后，流程、效率和产出发生了什么变化。
- 再总结值得推广的实践方法。
- 最后补充可能存在的问题、风险和适用边界。

建议约定：

- 文件名使用英文 slug，便于后续迁移和发布。
- 正式发布前可再移动到对应内容目录。
- 每篇草稿尽量包含现状、变化、实践、问题和总结五个部分。
